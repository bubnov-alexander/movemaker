# Быстрые кандидаты и прогресс CLI — спецификация

## Цель

Сократить время до первого готового ролика на CPU: тяжёлый анализ движения и звука должен выполняться только для ограниченного числа перспективных фрагментов, а CLI должен честно показывать текущий этап и счётчик прогресса.

## Поведение

После создания сцен и расшифровки конвейер строит все интервалы допустимой длительности, но рассчитывает для них только быстрые текстовую и длительную оценки. Затем он сохраняет `prefiltered_candidates.json` и выбирает не более `analysis_limit` лучших фрагментов для тяжёлой оценки.

Параметр CLI и YAML-конфигурации `analysis_limit` имеет значение по умолчанию 30. Он должен быть положительным целым числом. Тяжёлый этап не обрабатывает больше этого числа кандидатов.

Для каждого выбранного кандидата последовательно вычисляются метрики движения и звука. После завершения оценки отобранного набора выбирается лучший непересекающийся кандидат и немедленно рендерится как `short-01.mp4`. Затем создаются остальные ролики до заданного `count`.

## Прогресс

CLI пишет русскоязычные сообщения в stderr:

```text
[1/5] Проверка видео
[2/5] Поиск сцен
[3/5] Расшифровка речи
[4/5] Быстрый отбор: 30 кандидатов из 3412
[5/5] Точная оценка: 12/30
Создан первый ролик: shorts/short-01.mp4
```

Сообщения отражают только реально завершённые действия. Прогресс не должен печатать технический вывод FFmpeg или faster-whisper.

## Повторный запуск и ошибки

`prefiltered_candidates.json`, `candidates.json`, субтитры и готовые MP4 остаются в директории запуска. При повторе завершённые валидные этапы используются повторно.

CLI перехватывает `UserFacingError`, возникшие как при подготовке конфигурации, так и при выполнении `Pipeline`, печатает русское сообщение без traceback и завершает процесс с кодом 1 или 2 согласно типу ошибки.

## Вне объёма

- параллельная обработка кандидатов;
- графический прогресс-бар и веб-интерфейс;
- изменение формата итоговых роликов;
- повторная транскрибация уже сохранённого `transcript.json`.

## Критерии готовности

1. На 364 сценах тяжёлая оценка получает не более 30 кандидатов по умолчанию.
2. CLI показывает этапы и текущий номер при точной оценке.
3. После завершения точной оценки первым рендерится `short-01.mp4`.
4. Ошибочный путь к видео приводит к одной русской строке без Python traceback.
5. Все существующие тесты и новые тесты ограниченного отбора, прогресса и обработки ошибки проходят.
