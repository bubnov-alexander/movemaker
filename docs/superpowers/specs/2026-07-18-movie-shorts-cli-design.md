# Movie Shorts CLI — проектная спецификация MVP

## Цель

Создать полностью локальную CLI-утилиту, которая принимает одно русскоязычное видео и автоматически создаёт от трёх до пяти вертикальных кандидатов Shorts без ручного монтажа. Каждый итоговый ролик длится от 20 до 120 секунд и содержит синхронизированные русские субтитры.

MVP не использует платные API, облачные AI-сервисы, базу данных, очередь задач или веб-интерфейс.

## Пользовательский сценарий

```bash
movie-shorts create ./movie.mp4 \
  --output ./runs/film-001 \
  --count 5 \
  --min-duration 20 \
  --max-duration 120 \
  --device auto
```

Параметр `--device auto` выбирает CUDA при наличии совместимого NVIDIA GPU и использует CPU в остальных случаях. Язык транскрибации по умолчанию — русский (`ru`). Конфигурация и интерфейс транскрибатора допускают добавление английского языка, однако английский не входит в объём тестирования и поддержки MVP.

## Архитектура

Конвейер построен из независимо тестируемых модулей с JSON-совместимыми контрактами:

```text
source video
  -> MediaValidator
  -> SceneDetector
  -> Transcriber
  -> CandidateBuilder
  -> TextScorer + MotionScorer + AudioScorer + DurationScorer
  -> ScoreCalculator
  -> CandidateSelector
  -> SubtitleBuilder
  -> Renderer
  -> vertical shorts
```

### Модули

| Модуль | Назначение | Основные зависимости |
| --- | --- | --- |
| `MediaValidator` | Проверяет путь, контейнер, аудио- и видеопотоки; читает длительность. | ffprobe |
| `SceneDetector` | Определяет границы визуальных сцен. | PySceneDetect |
| `Transcriber` | Один раз распознаёт весь исходный фильм с сегментами и таймкодами слов. | faster-whisper |
| `CandidateBuilder` | Объединяет соседние сцены в смысловые интервалы 20–120 секунд. | scene и transcript metadata |
| `TextScorer` | Оценивает русские ключевые слова и плотность речи. | transcript metadata, настраиваемый словарь |
| `MotionScorer` | Оценивает динамику по разнице кадров в интервале. | OpenCV |
| `AudioScorer` | Оценивает среднюю громкость и пики. | FFmpeg astats |
| `DurationScorer` | Предпочитает интервалы 35–75 секунд внутри разрешённого диапазона. | candidate metadata |
| `ScoreCalculator` | Нормализует метрики, применяет веса и сохраняет расшифровку итоговой оценки. | scorer outputs |
| `CandidateSelector` | Выбирает до запрошенного числа непересекающихся кандидатов. | scored candidates |
| `SubtitleBuilder` | Создаёт ASS с крупным русским текстом и подсветкой текущего слова. | transcript words |
| `Renderer` | Вырезает диапазон из исходника, преобразует его в 9:16 и накладывает субтитры. | FFmpeg |

Исходное видео не режется на временные файлы при анализе. Вырезание и кодирование происходят только в `Renderer` для отобранных кандидатов.

## Оценка и отбор

Каждый кандидат содержит начало, конец, текст, значения оценок и итоговый балл. Базовая формула использует настраиваемые веса для текста, движения, аудио и длительности. Все значения нормализуются в диапазон 0–100 до расчёта итогового балла.

Кандидаты не должны пересекаться. Если качественных непересекающихся интервалов меньше `--count`, программа создаёт доступное число роликов и указывает это в отчёте.

MVP не пытается гарантировать полноценный драматургический анализ. Отбор является объяснимой эвристикой, которую позже можно дополнить локальным LLM-оценщиком через отдельный интерфейс, без изменения остального конвейера.

## Вертикальный формат и субтитры

Первый вариант вертикального формата: центральный crop в 9:16 с размытым фоном для сохранения кадра. Модуль рендеринга изолирован, поэтому позднее центральный crop может быть заменён на трекинг лица или объекта.

Субтитры создаются в ASS на основе пословных таймкодов faster-whisper. Стиль: крупный шрифт, контрастная обводка и выделение текущего слова. Если пословные таймкоды недоступны, формируются субтитры по сегментам без подсветки слова, а в отчёт добавляется предупреждение.

## Файлы результата

Каждый запуск создаёт отдельную директорию:

```text
runs/film-001/
  manifest.json
  scenes.json
  transcript.json
  candidates.json
  subtitles/
    short-01.ass
  shorts/
    short-01.mp4
  logs/
    pipeline.log
    debug.log
```

`manifest.json` хранит параметры запуска, версии инструментов и статусы этапов. `candidates.json` содержит объяснимые оценки всех кандидатов и причины, по которым были отобраны финальные интервалы.

## Обработка ошибок и перезапуск

Все пользовательские сообщения CLI и рекомендации по исправлению выводятся на русском языке. Технический вывод внешних инструментов сохраняется в `logs/debug.log`.

После каждого завершённого этапа обновляется `manifest.json`. При повторном запуске валидные результаты завершённых этапов используются повторно; незавершённые или невалидные этапы выполняются заново. Временный файл рендера получает финальное имя только после успешного завершения FFmpeg и проверки выходного видео.

Примеры пользовательских ошибок:

- «Не удалось открыть видео: проверьте путь и формат файла.»
- «CUDA недоступна. Обработка будет продолжена на CPU.»
- «Не удалось создать ролик №2. Технические детали сохранены в logs/debug.log.»

## Тестирование и критерии готовности

### Тесты

- Модульные: таймкоды, нормализация метрик, формирование непересекающихся кандидатов, генерация ASS.
- Интеграционные: короткое тестовое видео проходит через реальный FFmpeg и создаёт файл с ожидаемыми характеристиками.
- Ручная проверка: 3–5 русскоязычных видео разных жанров; проверяются читаемость субтитров, отсутствие обрывов и приемлемость выбора фрагментов.

### Критерии готовности MVP

1. Один русский фильм обрабатывается на CPU и NVIDIA GPU, если GPU доступен.
2. Создаются от трёх до пяти воспроизводимых вертикальных MP4 длительностью 20–120 секунд, если анализ нашёл достаточное число непересекающихся кандидатов.
3. Каждый итоговый ролик содержит синхронизированные русские ASS-субтитры.
4. Повторный запуск с одинаковыми видео и конфигурацией выдаёт одинаковые временные диапазоны.
5. Некорректное видео, ошибка FFmpeg и отсутствие CUDA не приводят к порче результата и выводят понятное русское сообщение.

## Вне объёма MVP

- веб-интерфейс, база данных и очередь задач;
- публикация в социальные сети;
- генерация озвучки или музыки;
- распознавание персонажей и имен;
- платные облачные сервисы и API;
- гарантированная семантическая оценка драматургии.

## Последующее развитие

После измерения качества MVP возможны: умный crop с трекингом лиц, английская транскрибация, SQLite/PostgreSQL, очередь задач, веб-интерфейс, локальный LLM-оценщик и генерация нескольких вариантов роликов для A/B-тестирования.
