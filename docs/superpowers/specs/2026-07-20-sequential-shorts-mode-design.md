# Режим последовательной нарезки Shorts

## Цель

Добавить второй режим генерации, который покрывает весь эпизод последовательными Shorts, сохраняя текущий режим отбора лучших моментов.

## CLI

Команда принимает `--mode highlights|sequential`.

- `highlights` — значение по умолчанию; текущий отбор лучших непересекающихся моментов, ограниченный `--count`.
- `sequential` — последовательная нарезка всего диапазона после `skip_intro` и до `skip_outro`; `--count` игнорируется.

## Границы частей

Целевая длина части — 60 секунд, допустимый диапазон — 45–75 секунд. Для каждой следующей части алгоритм выбирает ближайшую к 60 секундам границу сцены. При равных вариантах предпочтение получает граница после завершённой реплики. Если подходящей границы нет, используется 60 секунд. Последняя часть включает остаток диапазона, если он длиннее минимальной длительности.

## Совместимость

Рендер, субтитры, музыка, зеркалирование и трёхзональная компоновка не меняются. В manifest сохраняется выбранный режим. Все пользовательские сообщения остаются на русском.

## Проверка

Unit-тесты покрывают CLI-парсинг режима, построение последовательных интервалов, границы 45–75 секунд и игнорирование `count`. Pipeline-тест подтверждает, что sequential рендерит все построенные части.
