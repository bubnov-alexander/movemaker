# Movie Shorts

Локальная CLI-утилита, которая анализирует русскоязычное видео и собирает до пяти вертикальных роликов-кандидатов с ASS-субтитрами. Проект не использует платные API или облачные AI-сервисы.

## Требования

- Python 3.12+
- FFmpeg и ffprobe в `PATH`
- NVIDIA GPU — необязательно. Режим `--device auto` использует CUDA при наличии и CPU в остальных случаях.

## Установка

```bash
python3 -m venv .venv
.venv/bin/python -m pip install --upgrade pip
.venv/bin/python -m pip install -e '.[dev]'
```

## Запуск

```bash
movie-shorts create ./movie.mp4 --output ./runs/film-001 --count 5 --device cpu
movie-shorts create ./movie.mp4 --output ./runs/film-001 --device auto --config ./config.yaml
movie-shorts create ./movie.mp4 --output ./runs/film-001 --skip-outro 60
movie-shorts create ./episode.mp4 --output ./runs/episode-001 --mode sequential --skip-intro 90 --skip-outro 120
```

Для запуска на GPU после установки CUDA-зависимостей используйте `./gpu.sh` вместо `movie-shorts`: скрипт сам настроит пути к cuBLAS и cuDNN.

```bash
./gpu.sh create ./movie.mp4 --output ./runs/film-001 --count 5 --device auto
```

Параметры командной строки имеют приоритет над значениями `config.yaml`. Начните с [config.example.yaml](config.example.yaml).

`analysis_limit` (по умолчанию `30`) ограничивает число фрагментов, для которых считаются медленные метрики движения и звука. Чем меньше значение, тем быстрее появляется первый ролик, но тем меньше вариантов участвует в точном отборе. Во время работы CLI выводит этапы и счётчик точной оценки в stderr.

`skip_outro` (по умолчанию `60`) исключает финальные секунды фильма из кандидатов. Укажите `--skip-outro 0`, чтобы не пропускать outro, или задайте значение в `config.yaml`.

`--mode sequential` создаёт последовательные Shorts по всему эпизоду. Длина частей стремится к 60 секундам и может составлять от 45 до 75 секунд, чтобы закончиться на границе сцены; `--count` в этом режиме не применяется.

В результате создаются `manifest.json`, `scenes.json`, `transcript.json`, `candidates.json`, файлы `subtitles/*.ass`, итоговые `shorts/*.mp4` и технический журнал `logs/debug.log`.

## Фоновая музыка

Скопируйте треки в локальную (не добавляемую в Git) папку `music/` и укажите их пути в `config.yaml`: для эпичных сцен — YA YA, для остальных — altyn instrumental. Конвейер выбирает YA YA по совокупности текста, движения, энергии оригинального звука и общего балла сцены. Во время речи и громких моментов FFmpeg автоматически приглушает фон; оригинальная дорожка не меняется.

```yaml
background_music:
  epic_path: music/yaya.mp3
  calm_path: music/altyn.mp3
  max_volume: 0.12
  quiet_volume: 0.18
  epic_threshold: 60
```

Уменьшите `max_volume`, если фон заметен поверх диалогов. Увеличьте `epic_threshold`, если YA YA должен включаться только в самых сильных сценах.

Если нужен один трек для всех сцен, укажите только один из путей:

```yaml
background_music:
  calm_path: music/altyn.mp3
  max_volume: 0.12
  quiet_volume: 0.18
```

Отсутствующий путь автоматически использует этот же трек.

## Видео в верхней части

Для трёхзональной компоновки положите локальное видео в `backgrounds/background.mp4` и добавьте в `config.yaml`:

```yaml
layout_background_path: backgrounds/background.mp4
```

Видео зацикливается в верхней четверти кадра, фрагмент фильма занимает среднюю половину, а в нижней четверти виден размытый фон исходного видео. Звук видеофона не используется.

## Обработка ошибок

Сообщения CLI выводятся на русском. При ошибке FFmpeg или распознавания подробности сохраняются в `logs/debug.log`. При отсутствии CUDA обработка в режиме `auto` продолжается на CPU.

## Ограничения

Транскрибация полнометражного фильма требует значительного времени и места для модели; GPU заметно ускоряет этап Whisper. Эвристики выбора фрагментов не гарантируют драматургическое качество, поэтому итоговые ролики следует просматривать перед публикацией.

Перед использованием фильмов, музыки и других материалов убедитесь, что у вас есть необходимые права и что публикация соответствует правилам выбранной площадки.
